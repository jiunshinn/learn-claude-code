🔀 Git

Git 활용법

파일의 변경 이력을 관리하는 Git. 어렵게 느껴져도 Claude Code와 함께라면 쉬워요.

📓 비유로 이해하기

문서 작업을 할 때 "보고서_최종.docx", "보고서_최종_진짜최종.docx", "보고서_최종_수정_v3.docx"처럼 파일을 복사해본 적 있나요? Git은 이런 버전 관리를 자동으로 해주는 도구예요. 파일을 하나만 유지하면서도, 언제 누가 무엇을 바꿨는지 전부 기록해 줍니다. 실수로 파일을 망쳐도 이전 버전으로 쉽게 되돌릴 수 있어요.

🧩 핵심 개념 4가지

이것만 알면 Git의 80%를 이해한 거예요.

📁

저장소 (Repository)

Git이 변경 이력을 추적하는 프로젝트 폴더예요. git init 명령으로 만들거나, GitHub에서 가져올 수 있어요.

  • 프로젝트의 모든 파일과 변경 기록이 여기에
  • 로컬(내 컴퓨터)과 원격(GitHub)이 있음
💾

커밋 (Commit)

"이 시점을 기억해줘"라고 저장하는 거예요. 게임의 세이브 포인트와 같아요.

  • 변경 내용 + 설명 메시지를 함께 기록
  • 언제든 이 시점으로 되돌아갈 수 있음
🌿

브랜치 (Branch)

원본을 건드리지 않고 새로운 작업을 시도하는 "복사본 공간"이에요.

  • 실험적인 변경을 안전하게 시도 가능
  • 완성되면 원본에 합칠 수 있음 (Merge)
☁️

푸시 / 풀 (Push / Pull)

내 컴퓨터의 변경 사항을 GitHub에 올리거나(Push), GitHub의 변경 사항을 가져오는(Pull) 거예요.

  • Push: 내 작업을 팀과 공유
  • Pull: 다른 사람의 변경 사항을 받아오기

🤖 Claude Code가 Git을 쉽게 만들어줘요

복잡한 Git 명령어를 외울 필요 없어요. Claude에게 말로 시키면 돼요.

1

자동 커밋 메시지

/commit을 사용하면 Claude가 변경 내용을 분석해서 의미 있는 커밋 메시지를 자동으로 작성해 줘요.

2

자연어로 Git 조작

"새 브랜치 만들어줘", "마지막 변경 사항 되돌려줘" 같은 일상 표현으로 Git을 다룰 수 있어요.

3

충돌 해결 도우미

여러 사람이 같은 파일을 수정해서 충돌이 나면, Claude가 차이점을 분석하고 해결 방법을 알려줘요.

4

변경 이력 검색

"이 파일이 마지막으로 언제 수정됐어?", "이 부분을 왜 바꿨는지 알려줘" 같은 질문도 가능해요.

💬 이렇게 말해보세요

실제로 Claude Code에서 바로 사용할 수 있는 프롬프트예요.

🆕

Git 시작하기

새 프로젝트에 Git을 처음 설정할 때

프롬프트 이 폴더에 Git 저장소를 초기화하고, .gitignore 파일도 만들어줘. Node.js 프로젝트야.
💾

변경 사항 저장하기

작업한 내용을 커밋으로 기록할 때

프롬프트 지금까지 변경한 내용을 정리해서 커밋 메시지와 함께 저장해줘.
🔍

변경 이력 확인하기

어떤 파일이 언제, 왜 바뀌었는지 알고 싶을 때

프롬프트 최근 일주일간 이 프로젝트에서 변경된 파일 목록을 보여줘. 각 변경의 요약도 알려줘.
⏪

실수 되돌리기

잘못된 변경을 이전 상태로 복구할 때

프롬프트 방금 수정한 config.json 파일을 수정 전 상태로 되돌려줘.
🌿

브랜치로 안전하게 작업하기

원본을 유지하면서 새로운 기능을 시도할 때

프롬프트 "new-design"이라는 브랜치를 만들어줘. 여기서 스타일을 실험해볼 거야.
☁️

GitHub에 올리기

내 작업을 GitHub 원격 저장소에 공유할 때

프롬프트 지금까지 작업한 내용을 GitHub에 푸시해줘.

💡 비개발자를 위한 팁

Git 명령어를 직접 외울 필요는 전혀 없어요! Claude Code에게 하고 싶은 일을 말로 설명하면, Claude가 알아서 올바른 Git 명령어를 실행해 줍니다. "커밋해줘", "되돌려줘", "브랜치 만들어줘"처럼 편하게 말하세요. 단, Claude가 실행하기 전에 무엇을 할지 설명해 줄 거예요 — 잘 읽어보고 승인하면 돼요.
