자주 묻는 질문

네, 가능합니다! Claude Code에서 하는 대화는 한국어로 자유롭게 하면 돼요. '이 파일 정리해줘', '보고서 만들어줘' 같은 일상적인 말로 충분합니다. 터미널 명령어도 Claude가 대신 실행해 주기 때문에 외울 필요가 없어요.
Claude Pro 구독 (월 $20)이면 시작할 수 있어요. 더 많이 사용하려면 Max 5x ($100/월) 또는 Max 20x ($200/월) 플랜도 있어요. 가벼운 사용이라면 Pro로 충분합니다. API 키를 직접 구매하는 방법도 있지만, 비개발자에게는 구독이 가장 간편해요.
Claude Code는 여러분의 로컬 컴퓨터에서 동작해요. 파일 내용은 Claude API로 전송되어 처리되지만, Anthropic은 Pro/Max 구독 사용자의 데이터를 모델 학습에 사용하지 않습니다. 그래도 민감한 정보(비밀번호, 개인정보 등)는 별도로 관리하는 것이 좋아요.
가장 큰 차이는 파일 접근이에요. 웹 Claude는 파일을 업로드해야 하지만, Claude Code는 폴더의 모든 파일에 직접 접근합니다. 또한 파일을 직접 생성·수정·삭제할 수 있고, 대화 길이 제한도 훨씬 넉넉해요. 반복 작업 자동화와 대용량 데이터 처리에 특히 강합니다.
Claude Code는 파일을 수정하기 전에 항상 허락을 구해요. 변경 내용을 미리 보여주니까 확인 후 승인하면 돼요. 만약 실수가 발생했다면, Esc를 두 번 누르면 이전 상태로 되돌릴 수 있는 체크포인트가 표시됩니다. 중요한 파일은 작업 전에 백업해 두는 것도 좋은 습관이에요.
기본적으로 Claude Code는 로컬 파일 작업에 집중해요. 하지만 웹 검색이나 URL 접근 기능을 추가할 수 있고, MCP(Model Context Protocol) 서버를 연결하면 Slack, GitHub, 데이터베이스 등 외부 서비스와도 연동할 수 있어요.
네! Windows, Mac, Linux 모두 지원해요. Windows에서는 먼저 Git for Windows를 설치한 후, PowerShell이나 명령 프롬프트(CMD)에서 사용할 수 있어요. 터미널이 어려우시면 데스크톱 앱을 다운받아서 일반 프로그램처럼 사용하거나, claude.ai/code에서 웹 브라우저로 바로 사용할 수도 있어요.
네, 여러 방법이 있어요!

🖥️ 데스크톱 앱: 일반 프로그램처럼 설치해서 사용. 시각적으로 변경사항을 확인하기 좋아요.
🌐 웹 버전: claude.ai/code에서 설치 없이 브라우저로 사용. 긴 작업을 걸어두고 나중에 확인할 수 있어요.
📝 VS Code: VS Code 에디터 안에서 확장 프로그램으로 사용.
📱 모바일: Claude iOS 앱에서도 Code 기능을 사용할 수 있어요.

모든 환경에서 CLAUDE.md 설정이 공유되니, 어디서 시작하든 같은 경험이에요.

🙋 팀원 Q&A

팀원들이 실제로 궁금해한 질문과 답변이에요.

Claude Code에서 GitHub를 관리하는 핵심 흐름이에요:

1. 저장소 연결
GitHub에서 저장소를 만들고, Claude Code에게 "이 폴더를 GitHub 저장소와 연결해줘"라고 말하면 됩니다.

2. 일상적인 관리
• /commit → 변경 사항을 의미 있는 메시지와 함께 저장
• "GitHub에 푸시해줘" → 원격에 동기화
• "PR 만들어줘" → Pull Request를 자동 생성

3. 브랜치 전략
• main 브랜치는 완성된 버전만 유지
• 새 작업마다 브랜치를 만들어서 진행 → 완료 후 PR로 합치기
• Claude에게 "feature/새기능 브랜치 만들어줘"라고 말하면 끝

4. 유용한 습관
• 작은 단위로 자주 커밋하기 (하루에 한 번보다 기능 단위로)
• 커밋 메시지는 Claude가 자동 생성해주니 /commit 활용
• .gitignore로 민감한 파일(비밀번호, API 키)은 제외
CLI(Claude Code)와 GitHub가 만나면 강력한 협업 워크플로우가 됩니다:

🔄 기본 협업 흐름
① 각자 브랜치에서 작업 → ② Claude Code로 커밋/푸시 → ③ GitHub에서 PR 생성 → ④ 리뷰 후 머지

📌 핵심 협업 포인트
• PR(Pull Request)이 협업의 중심이에요. Claude에게 "PR 만들어줘"라고 하면 변경 내용 요약까지 자동으로 작성해줘요.
• 코드 리뷰: GitHub PR에 댓글을 달아 서로 피드백. Claude에게 "이 PR 리뷰해줘"라고 요청하면 코드 분석도 가능해요.
• 충돌 해결: 같은 파일을 동시에 수정하면 충돌이 나는데, Claude에게 "충돌 해결해줘"라고 말하면 도와줘요.

🎯 비개발자 팁
• GitHub의 Issues 탭을 할일 목록처럼 활용하세요
• PR 설명에 "왜 이 변경을 했는지"를 적으면 팀 소통이 매끄러워져요
• Claude Code에서 gh 명령어로 이슈 생성, PR 확인 등 GitHub 작업을 터미널에서 바로 할 수 있어요
AI 도구마다 강점이 달라요. 상황별로 최적의 도구를 정리해 봤어요:

📝 문서/글 작성 → 웹 Claude (claude.ai)
보고서, 이메일, 기획안 등 텍스트 중심 작업. 파일 업로드로 분석도 가능.

🛠️ 파일 작업/자동화 → Claude Code
여러 파일을 동시에 처리, 반복 작업 자동화, 코드 생성/수정, 프로젝트 관리. 로컬 파일에 직접 접근 가능한 것이 핵심 차별점.

🎨 디자인/시각 자료 → Cursor, v0, Figma AI
UI/UX 디자인, 프로토타이핑, 시각적 결과물이 필요할 때.

🔍 리서치/정보 수집 → ChatGPT, Perplexity
최신 정보 검색, 시장 조사, 경쟁사 분석 등 웹 검색이 중요한 작업.

💡 선택 기준 요약
• "파일을 직접 만들거나 수정해야 한다" → Claude Code
• "대화하며 아이디어를 정리하고 싶다" → 웹 Claude / ChatGPT
• "시각적 결과물이 필요하다" → Cursor / v0
• "최신 정보를 찾아야 한다" → Perplexity / ChatGPT
• 하나만 고르기 어려우면 Claude Code + 웹 Claude 조합이 가장 범용적이에요
아무것도 없는 상태에서 프로젝트를 시작하는 "0 to 1" 바이브코딩 팁이에요:

1단계: 기획을 먼저 잡기
코드보다 먼저 CLAUDE.md에 프로젝트의 목적, 대상 사용자, 핵심 기능을 정리하세요. Claude에게 "이 아이디어를 CLAUDE.md로 정리해줘"라고 요청하면 돼요.

2단계: 작게 시작하기
처음부터 완벽한 걸 만들려 하지 마세요. 핵심 기능 하나만 먼저 동작하게 만드세요.
• 예: 웹사이트 → 메인 페이지 1개부터
• 예: 자동화 → 가장 반복적인 작업 1개부터

3단계: 대화하듯 만들기
Claude에게 한 번에 모든 걸 시키지 마세요. 작은 단위로 요청 → 확인 → 다음 단계를 반복하세요.
• ❌ "쇼핑몰 전체를 만들어줘"
• ✅ "상품 목록 페이지 먼저 만들어줘" → "장바구니 기능 추가해줘" → ...

4단계: 자주 저장하기
잘 되는 시점마다 /commit으로 저장하세요. 문제가 생기면 되돌릴 수 있어요.

5단계: 레퍼런스 활용
"이 사이트처럼 만들어줘"라고 URL이나 스크린샷을 보여주면 Claude가 빠르게 방향을 잡아줘요. 말로 설명하기 어려운 디자인이나 기능은 시각적 레퍼런스가 효과적이에요.
Claude Code에는 자동 체크포인트 기능이 있어서 변경 이력을 쉽게 관리할 수 있어요:

🔖 체크포인트란?
Claude Code가 파일을 수정할 때마다 자동으로 "이전 상태"를 기록해 둡니다. 게임의 자동 저장과 비슷해요.

⏪ 되돌리는 방법
• Esc를 두 번 누르면 → 최근 체크포인트 목록이 나타남
• 원하는 시점을 선택하면 → 그 상태로 파일이 복원됨
• 체크포인트는 대화 중에 자동 생성되므로 별도 설정 불필요

📋 기획자를 위한 활용 워크플로우
① 기획자가 Claude Code로 수정 사항 요청 (예: "이 페이지 문구를 이렇게 바꿔줘")
② Claude가 수정 → 자동으로 체크포인트 생성
③ 결과 확인 후 마음에 안 들면 → Esc × 2로 되돌리기
④ 마음에 들면 → /commit으로 확정 저장

💡 더 체계적으로 관리하려면
• Git 커밋을 단계마다 생성: 수정 완료될 때마다 /commit을 실행하면, 나중에 "최근 커밋 이력 보여줘"로 변경 내역을 한눈에 확인 가능
• 브랜치 활용: 기획자용 브랜치(예: planning/홈페이지-수정)를 따로 만들면, 개발자가 나중에 PR을 통해 변경 내용을 한 번에 검토 가능
• 커밋 메시지에 의도 남기기: Claude가 자동 생성한 커밋 메시지 외에도, "커밋 메시지에 '기획팀 요청: 메인 문구 변경'이라고 적어줘"라고 요청하면 나중에 누가 왜 바꿨는지 바로 알 수 있어요
• "이 파일의 최근 변경 이력 보여줘" 또는 "지난 3일간 바뀐 파일 목록 알려줘"로 히스토리를 쉽게 조회할 수 있어요
